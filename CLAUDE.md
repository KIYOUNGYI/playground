# 대화 기록

- 클로드 코드로 질의하면 /docs/history/{yymmdd-숫자}.md 파일에 질의한 내용/답변 내용을 기록할 것.
- 기록은 위에서 아래로 할 것
- 파일 이름의 날짜는 **오늘 날짜** 기준으로 생성한다. 이전 날짜 파일에 이어 쓰지 않는다.
  - ex> 오늘이 260515이면 260515-01.md 를 새로 생성
- 파일 줄 길이가 300줄 - 350줄에서 넘어갈 것 같으면 다음 파일 생성 할 것
  - ex> 260514-01.md -> 260514-02.md

- 매 대화 시작 시 프로젝트 루트에 `.recording-disabled` 파일 존재 여부를 확인한다.
  - 파일이 존재하면 → 기록하지 않는다.
  - 파일이 없으면 → 기록한다.

- 기록을 끄고 싶을 때: 사용자가 아래 문구를 입력하면 `.recording-disabled` 파일을 생성한다.
  - ex> 지금부터 대화는 기록 안해도 된다.
- 기록을 다시 켜고 싶을 때: 사용자가 아래 문구를 입력하면 `.recording-disabled` 파일을 삭제한다.
  - ex> 지금부터 대화는 다시 기록해줘.

### Use Case

| 상황 | 사용자 입력 | Claude 동작 |
|------|------------|-------------|
| 민감한 내용 논의 (보안, 개인정보 등) | `지금부터 대화는 기록 안해도 된다.` | `.recording-disabled` 생성, 이후 대화 기록 중단 |
| 단순 탐색 / 임시 질의 | `지금부터 대화는 기록 안해도 된다.` | 동일하게 기록 중단 |
| 기록이 다시 필요할 때 | `지금부터 대화는 다시 기록해줘.` | `.recording-disabled` 삭제, 이후 대화 기록 재개 |
| 기록 상태 확인 | `.recording-disabled` 파일 유무 확인 | 파일 있으면 OFF, 없으면 ON |
