**분석 내용:**

**image59 (4.1 구조화된 출력 변환기 — 개념 + 다이어그램):**
- 구조화된 출력(Structured Output): 데이터 의미와 관계를 고려한 JSON 형식 출력
- LLM 출력은 기본적으로 텍스트 → 형식 지침을 프롬프트에 포함해야 구조화 출력 가능
- `StructuredOutputConverter<T>` 인터페이스: extends `FormatProvider`, `Converter<String, T>`
- 아키텍처: Raw Input → PromptTemplate+FormatProvider → ChatModel → Raw Output → convert() → Structured Output(T)

**image60 (4.1 — 변환기 목록 및 상속도):**
- `FormatProvider.getFormat()`: 출력 형식 지침 문자열 반환
- `Converter<String, T>.convert()`: LLM 텍스트 → T 타입 변환
- 구현체: `ListOutputConverter`, `BeanOutputConverter<T>`, `MapOutputConverter`
- 사용 방법: 저수준(직접 생성), 고수준(`entity()` 체이닝)

**image61 (4.2 List(String)으로 변환 — ListOutputConverter LowLevel):**
- 저수준: `new ListOutputConverter()` → `converter.getFormat()` → PromptTemplate `{format}` 바인딩 → `.call().content()` → `converter.convert(commaSeparatedString)`

**image62 (4.2 — HighLevel + 컨트롤러):**
- 고수준: `.call().entity(new ListOutputConverter())` 체이닝
- 컨트롤러: `@PostMapping("/list-output-converter")`, `List<String>` 반환

**작성 결과:**
`chapter-4.md` 신규 생성. 4.1 개념 + 인터페이스 + 4.2 ListOutputConverter(저수준/고수준/컨트롤러) 작성 완료.
