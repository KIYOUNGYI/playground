## Q5. Q2의 1번 항목 설명

**질의:** `@docs/history/260514-01.md 의 Q2 에서 1번 내용에 대해 설명해줘.`

**답변:**

Q2의 1번 항목은 **Claude Code 슬래시 커맨드**입니다.

### Claude Code 슬래시 커맨드란?

Claude Code CLI에서 `/명령어` 형태로 실행하는 커스텀 명령어입니다.

예시:
```
/ocpp1.6 StartTransaction 스펙 알려줘
```

이 프로젝트에서는 `/ocpp1.6` 같은 커맨드를 만들어, OCPP 관련 질문을 AI가 내부 문서 기반으로 답변하도록 하는 것이 목표입니다.

### 어떻게 만드나요?

슬래시 커맨드는 `.claude/commands/` 디렉토리에 마크다운 파일로 정의합니다.

```
.claude/
  commands/
    ocpp1.6.md   ← /ocpp1.6 커맨드 정의
```

`ocpp1.6.md` 파일 안에 프롬프트 내용을 작성하면, `/ocpp1.6` 호출 시 그 프롬프트가 실행됩니다.

```markdown
OCPP 1.6 전문가로서 다음 질문에 답해주세요: $ARGUMENTS
```

`$ARGUMENTS`는 커맨드 뒤에 입력한 텍스트가 자동으로 들어가는 변수입니다.

### CLAUDE.md와의 관계

`CLAUDE.md`는 프로젝트 루트에 두는 설정 파일로, Claude Code가 해당 프로젝트에서 어떻게 동작할지 지시합니다. 슬래시 커맨드의 동작 방식이나 규칙을 여기에 정의할 수 있습니다.

**요약:** 1번을 먼저 배우는 이유는, 이 프로젝트의 핵심 진입점(사용자 인터페이스)이 슬래시 커맨드이기 때문입니다. 나머지 RAG, Java AI 등은 이 커맨드 안에서 호출되는 백엔드 로직입니다.
