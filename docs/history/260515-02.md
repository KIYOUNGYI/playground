## Q8. Java 21 / Spring Boot / Gradle 버전으로 RAG Hello World 가능한가?

**질의:** `Q2. Java 21 / SpringBoot / Gradle 버전으로는 Hello World 가능한가?`

**답변:**

가능하다. **Spring AI**가 Spring Boot 공식 RAG 솔루션이다.

**build.gradle**
```groovy
plugins {
    id 'java'
    id 'org.springframework.boot' version '3.3.5'
    id 'io.spring.dependency-management' version '1.1.6'
}

java {
    toolchain {
        languageVersion = JavaLanguageVersion.of(21)
    }
}

repositories {
    mavenCentral()
    maven { url 'https://repo.spring.io/milestone' }
}

ext {
    set('springAiVersion', '1.0.0')
}

dependencyManagement {
    imports {
        mavenBom "org.springframework.ai:spring-ai-bom:${springAiVersion}"
    }
}

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter'
    implementation 'org.springframework.ai:spring-ai-anthropic-spring-boot-starter'
    implementation 'org.springframework.ai:spring-ai-transformers-spring-boot-starter'
}
```

**application.yml**
```yaml
spring:
  ai:
    anthropic:
      api-key: ${ANTHROPIC_API_KEY}
      chat:
        options:
          model: claude-sonnet-4-6
```

**RagHelloWorldApplication.java**
```java
@SpringBootApplication
public class RagHelloWorldApplication implements CommandLineRunner {

    @Autowired private ChatClient.Builder chatClientBuilder;
    @Autowired private EmbeddingModel embeddingModel;

    public static void main(String[] args) {
        SpringApplication.run(RagHelloWorldApplication.class, args);
    }

    @Override
    public void run(String... args) {
        // 1. 문서 준비
        List<Document> documents = List.of(
            new Document("BootNotification은 충전기가 부팅될 때 중앙 시스템에 전송하는 첫 번째 메시지다."),
            new Document("Heartbeat 메시지는 충전기가 살아있음을 알리기 위해 주기적으로 전송된다."),
            new Document("RemoteStartTransaction은 중앙 시스템이 충전을 원격으로 시작할 때 사용한다."),
            new Document("StatusNotification은 충전기 상태 변화를 중앙 시스템에 알린다."),
            new Document("Authorize는 RFID 카드 인증을 처리하는 메시지다.")
        );

        // 2. 인메모리 Vector Store에 적재
        SimpleVectorStore vectorStore = SimpleVectorStore.builder(embeddingModel).build();
        vectorStore.add(documents);

        // 3. 사용자 질문
        String userQuery = "충전기가 처음 켜졌을 때 무슨 메시지를 보내?";

        // 4. 유사 문서 검색
        List<Document> similar = vectorStore.similaritySearch(
            SearchRequest.builder().query(userQuery).topK(2).build()
        );

        String context = similar.stream()
            .map(Document::getFormattedContent)
            .collect(Collectors.joining("\n"));

        System.out.println("=== 검색된 문서 ===");
        similar.forEach(d -> System.out.println("- " + d.getFormattedContent()));

        // 5. [문서 + 질문] → Claude 호출
        String prompt = """
            아래 참고 문서를 바탕으로 질문에 답하세요.

            [참고 문서]
            %s

            [질문]
            %s
            """.formatted(context, userQuery);

        String response = chatClientBuilder.build().prompt(prompt).call().content();
        System.out.println("\n=== Claude 답변 ===");
        System.out.println(response);
    }
}
```

**LangChain4j vs Spring AI 비교**

| | LangChain4j (순수 Java) | Spring AI (Spring Boot) |
|---|---|---|
| 진입장벽 | 낮음, 의존성 3개 | 높음, Spring 생태계 이해 필요 |
| 설정 | 코드로 직접 구성 | application.yml로 자동 구성 |
| 확장성 | 라이브러리 교체 자유 | Spring Bean 교체 방식 |
| 실제 서비스 | 단독 앱에 적합 | REST API / 서비스 서버에 적합 |

Hello World 목적이라면 LangChain4j가 더 단순하다. Spring Boot가 필요한 이유는 보통 RAG를 REST API로 서빙할 때다.

## Q10. RAG 흐름 재확인 — "DB에 질문해서 재가공해서 Claude에 질문한다"가 맞나?

**질의:** `지금 정리하면, 사용자가 질의하면 claude에 질의하기 전에 디비에 질문해서 질문을 재 가공해서 claude에 질문한다? 이게 맞아?`

**답변:**

거의 맞다. 보정할 부분만 짚으면:

```
사용자 질문
    ↓
① 질문을 벡터(숫자)로 변환
    ↓
② Vector DB에서 유사 문서 검색  ← "DB에 질문"하는 단계
    ↓
③ [원본 질문 + 검색된 문서] 조합  ← "재가공" 단계
    ↓
④ Claude 호출
```

- "DB에 질문한다" → 질문을 벡터로 바꾼 뒤 유사 문서를 검색하는 것
- "질문을 재가공" → 질문 자체가 바뀌는 게 아니라, [원본 질문 + 문서]를 합쳐 새 프롬프트를 만드는 것. 원본 질문은 그대로 Claude에게도 전달된다.

## Q12. 벡터 개념을 모른다. 어디서부터 시작해야 하나?

**질의:** `근데 정작 나는 벡터에 대한 개념을 잘 몰라. 수학 벡터 개념도 난 배운 적이 없어. 어디서부터 시작해야해?`

**답변:**

수학 없이 직관부터 잡으면 된다.

**벡터 = 숫자 목록.** `[8, 2, 5]` 이게 전부다.

영화 예시로 직관 설명:
```
          액션  로맨스  코미디
어벤저스 = [ 9,    1,     3 ]
토르     = [ 8,    2,     4 ]   ← 어벤저스와 가까움
노트북   = [ 1,    9,     2 ]   ← 어벤저스와 멀음
```
의미가 비슷하면 벡터가 비슷하고, 벡터가 비슷하면 거리가 가깝다. Embedding도 이 원리다.

배울 순서: 벡터 = 숫자 목록 → 거리 = 유사도 → Embedding = 텍스트→벡터 변환 (라이브러리가 해줌) → 코사인 유사도는 필요할 때.

RAG 구현에 수학은 필요 없다. 라이브러리가 전부 처리한다.

## Q14. API 요금은 한 달에 얼마?

**질의:** `1달에 돈 얼마 내야해?`

**답변:**

월정액 아님. 사용한 만큼 내는 종량제 (토큰 단위 과금).

- 토큰 = 텍스트 조각. 한글 1글자 ≈ 1~2토큰.
- 입력 토큰(프롬프트) + 출력 토큰(응답) 합산 청구.
- Hello World / 학습 수준은 무료 크레딧으로 충당 가능.
- 정확한 단가는 `anthropic.com/pricing` 에서 확인 (변동 있음).
