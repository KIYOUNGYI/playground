## Q. ./gradlew clean build 실행 시 permission denied 에러

`/src/this-is-ai/ch01-spring-ai-project`에서 `./gradlew clean build` 실행 시 permission denied 발생. 다른 워크스페이스(IntelliJ 로드)에서는 정상 동작함. 터미널에서 `./gradlew bootRun`은 되어야 한다는 요건.

## A.

`gradlew` 파일에 실행 권한(`x`)이 없는 것이 원인. 다른 워크스페이스에서 복사 또는 git으로 가져올 때 유실된 케이스.

아래 명령으로 해소:

```bash
# 단일 적용
chmod +x src/this-is-ai/ch01-spring-ai-project/gradlew

# 앞으로 챕터 추가 시 일괄 적용
find src/ -name gradlew -exec chmod +x {} \;
```

`find` 명령으로 `src/` 하위 전체 `gradlew`에 일괄 적용 완료 (`-rwxrwxrwx` 확인).

## Q. image4.jpeg 분석 요청

`docs/book/this-is-ai/image4.jpeg` 이미지를 분석해달라는 요청.

## A.

책 42페이지, **2.1 Chat Model API** 섹션 이미지 분석.

**모델 분류표:**

| 모델 구분 | 설명 | 주요 API |
|----------|------|---------|
| Chat Model | Text, Image To Text 모델 (LLM) | `ChatModel` |
| Image Model | Text To Image 모델 | `ImageModel` |
| Audio Model | Text To Speech, Speech To Text | `SpeechModel` |
| Embedding Model | Text To Vector 모델 | `EmbeddingModel` |

**핵심 내용:**
- Spring AI는 AI 모델을 같은 분류에 속하는 모델들을 통일화된 방법으로 사용하기 위해 API를 제공.
- 개발자는 업체별 제공 API에 종속되지 않고 이들 API를 이용해서 모델을 호출하고, 애플리케이션의 소스 변경 최소화 가능.

**다이어그램 구조:**
```
AI Application (Spring AI)
├── ChatModel  (A사 모델, B사 모델)
├── ImageModel (A사 모델, B사 모델)
└── EmbeddingModel (A사 모델, B사 모델)
```

Spring AI의 **벤더 중립성(Vendor Neutrality)** 설계 철학을 보여주는 그림. 표준 인터페이스를 제공하여 A사/B사 모델 교체 시 애플리케이션 코드 변경 최소화.
